How MonadKit works — prompt to payment

Two phases. Build time happens once, in the AI builder. Runtime happens every day, in the user's browser — with zero gas and zero crypto words.

▸ PHASE 1 · BUILD TIME — happens once
A creator describes the app. The AI builder consults MonadKit instead of guessing.

▸ PHASE 2 · RUNTIME — every user, every payment
The user taps "Pay $20". Five things happen in ~2 seconds. They see none of them.

Paymaster signs "I'll cover gas". Bundler wraps it in a real transaction and pays the MON itself. Free on testnet.

→
MONAD · PARALLEL EVM

🟣 Onchain in ~1s

EntryPoint verifies both signatures → smart account sends USDC transfer (or deploys a contract via CREATE2). Done.

BACK IN THE APP

✅ "Paid $20"

Share marked paid, txHash stored in the entity. User saw a spinner for a second. That's it. That's the demo.

WHO PAYS GAS? Never the user. Pimlico's bundler fronts the MON and its paymaster reimburses it — free on Monad Testnet, billed to the developer on mainnet. User balance required: 0 MON, forever.
WHY IT'S SAFE MonadKit is read-only at build time — it serves docs, config and code. It holds no keys, touches no funds, and isn't in the runtime path at all. Kill the MCP and every shipped app keeps working.
CUSTOM CONTRACTS? The MCP compiles Solidity server-side (solc inside the MCP). The user's own browser deploys the bytecode gas-free through the CREATE2 factory. No Remix, no CLI, no MetaMask.
